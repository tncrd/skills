# Landing Page Patterns

Choose based on product type and conversion goal.

## Content-first patterns

**Hero + Features + CTA**
Best for: SaaS, apps with clear value proposition.
Sections: Hero with headline + CTA → 3–4 feature blocks → social proof → final CTA.
CTA: above fold, repeated after features.

**Minimal Single Column**
Best for: simple products, apps, personal brands.
Sections: Hero → brief description → single CTA.
CTA: centered, above fold. One action only.

**Newsletter / Content First**
Best for: blogs, writers, newsletters, content creators.
Sections: Value proposition → sample content → subscribe CTA.
CTA: email input above fold, repeated at bottom.

## Trust-driven patterns

**Hero + Testimonials + CTA**
Best for: services, B2C, consulting.
Sections: Hero → testimonials (3–5) → features → CTA.
CTA: after social proof (trust before ask).

**Trust & Authority + Conversion**
Best for: B2B, enterprise, consulting, legal.
Sections: Hero → logos/clients → case studies → features → CTA.
CTA: after trust signals. May use "Book a demo" instead of "Sign up".

**Before-After Transformation**
Best for: fitness, coaching, SaaS with visible results.
Sections: Problem → solution → transformation proof → CTA.
CTA: after transformation evidence.

## Product-led patterns

**Product Demo + Features**
Best for: software, tools with visual UI.
Sections: Hero with product screenshot/video → feature deep-dive → pricing → CTA.
CTA: "Try free" above fold, pricing CTA below.

**Feature-Rich Showcase**
Best for: complex SaaS, multi-feature platforms.
Sections: Hero → feature grid (bento or detailed blocks) → integrations → pricing → CTA.
CTA: multiple entry points for different features.

**Interactive Product Demo**
Best for: developer tools, configurators, visual editors.
Sections: Hero → embedded interactive demo → features → CTA.
CTA: after user has experienced the product.

## Conversion-focused patterns

**Funnel (3-Step Conversion)**
Best for: lead generation, onboarding.
Sections: Hero → Step 1/2/3 → CTA.
CTA: progressive — each step narrows commitment.

**Pricing Page + CTA**
Best for: SaaS, subscription products.
Sections: Hero → pricing table (3 tiers typical) → FAQ → CTA.
CTA: per-tier buttons. Highlight recommended tier.

**Comparison Table + CTA**
Best for: products competing against alternatives.
Sections: Hero → comparison table → differentiators → CTA.
CTA: after user sees competitive advantage.

## Special-purpose patterns

**Waitlist / Coming Soon**
Sections: Hero with value prop → email capture → optional: team/vision.

**App Store Style**
Sections: Hero with device mockup → feature highlights → ratings → download buttons.

**Event / Conference**
Sections: Hero with date/location → speakers → schedule → register CTA.

**Portfolio Grid**
Sections: Brief intro → filterable project grid → contact CTA.

**Bento Grid Showcase**
Sections: Hero → bento grid of features (mixed sizes) → CTA.
Best for: modern SaaS, product features, personal dashboards.

## Recommended combinations by industry

| Industry | Pattern | Style direction |
|----------|---------|----------------|
| SaaS | Hero + Features + CTA | Clean, spacious, demo-forward |
| E-commerce | Feature-Rich or Product Demo | Visual, product-centric, urgency |
| Portfolio | Minimal Single Column or Portfolio Grid | Personal, distinctive, content-forward |
| Blog/Newsletter | Newsletter / Content First | Editorial, readable, subscribe-focused |
| B2B / Enterprise | Trust & Authority | Professional, case-study-driven |
| Developer tool | Interactive Product Demo | Technical, show-don't-tell |
| Startup (pre-launch) | Waitlist / Coming Soon | Bold, aspirational, email capture |
